    Recherche: CLI Handling¶

    Überblick zu Bibliotheken für Kommandozeilen-Argumente in Python.

    argparse (Standardbibliothek)¶

    Teil von Python, keine Installation nötig.

    import argparse

parser = argparse.ArgumentParser(description="Dateien verarbeiten")
parser.add_argument("datei", help="Pfad zur Eingabedatei")
parser.add_argument("-o", "--output", default="out.txt")
parser.add_argument("-v", "--verbose", action="store_true")

args = parser.parse_args()

    Aufruf: python skript.py input.csv -o result.csv -v

    Typ-Prüfung, Auswahl, Pflichtfeld, mehrere Werte:

    parser.add_argument("--anzahl", type=int, default=10)
parser.add_argument("--modus", choices=["dev", "prod"])
parser.add_argument("--rate", type=float, required=True)
parser.add_argument("--tags", nargs="+")  # eine oder mehrere Tags

    Subcommands (z. B. mytool add foo, mytool list):

    sub = parser.add_subparsers(dest="command")
p_add = sub.add_parser("add")
p_add.add_argument("item")
sub.add_parser("list")

    Bei falschem Aufruf bricht das Programm automatisch ab und gibt einen Hinweis aus:

    $ python skript.py
usage: skript.py [-h] datei
skript.py: error: the following arguments are required: datei
# Exit-Code 2

    Vorteile - Immer verfügbar, keine zusätzliche Abhängigkeit - Help-Text (-h) wird automatisch erzeugt - Subcommands und eigene Validierung möglich

    Nachteile - Viel Schreibarbeit pro Argument - Keine Type Hints, kein Lesen von Umgebungsvariablen - Keine Farben, keine Prompts

    click (extern)¶

    Installation: pip install click · https://github.com/pallets/click

    Nutzt Decorators. Funktion wird zum Befehl.

    import click

@click.command()
@click.argument("datei")
@click.option("-o", "--output", default="out.txt")
@click.option("-v", "--verbose", is_flag=True)
def verarbeiten(datei, output, verbose):
    """Verarbeitet DATEI und schreibt nach OUTPUT."""
    click.echo(f"Lese {datei}, schreibe nach {output}")

if __name__ == "__main__":
    verarbeiten()

    Eingebaute Typen prüfen automatisch (Datei muss existieren, Wert muss in Range sein, …):

    @click.option("--datei", type=click.Path(exists=True, readable=True))
@click.option("--log", type=click.File("w"))  # öffnet/schließt selbst
@click.option("--modus", type=click.Choice(["dev", "prod"]))
@click.option("--prozent", type=click.IntRange(0, 100))

    Umgebungsvariablen lesen: Statt os.environ["MY_KEY"] reicht ein Parameter.

    @click.option("--key", envvar="MY_KEY")
# export MY_KEY=abc → wird automatisch übernommen

    Interaktive Eingabe (z. B. Passwort): click fragt nach, wenn nichts übergeben wurde.

    @click.option("--passwort", prompt=True, hide_input=True)
# Aufruf:  Passwort: ******

    Subcommands über @click.group() (lassen sich auch verschachteln, z. B. tool user add):

    @click.group()
def cli(): pass

@cli.command()
def add(): click.echo("hinzugefügt")

@cli.command()
def list(): click.echo("liste")
# Aufruf: python tool.py add

    Test-Runner CliRunner startet den Befehl im Test ohne echten Shell-Aufruf:

    from click.testing import CliRunner
result = CliRunner().invoke(cli, ["add"])
assert result.exit_code == 0

    Vorteile - Wenig Boilerplate, Docstring wird zum Help-Text - Eingebaute Typen, Prompts, Farben, Umgebungsvariablen - Gut testbar, ausgereifte Bibliothek

    Nachteile - Viele Decorators bei großen Befehlen unübersichtlich - Type Hints werden nicht genutzt, Typ muss in type=... doppelt angegeben werden - Etwas langsamerer Start als argparse

    typer (extern)¶

    Installation: pip install typer · https://github.com/fastapi/typer

    Baut auf click auf. Nutzt Type Hints zur Konfiguration.

    import typer

app = typer.Typer()

@app.command()
def verarbeiten(datei: str, output: str = "out.txt", verbose: bool = False):
    """Verarbeitet DATEI und schreibt nach OUTPUT."""
    typer.echo(f"Lese {datei}, schreibe nach {output}")

if __name__ == "__main__":
    app()

    Aufruf: python skript.py input.csv --output result.csv --verbose

    Typ, Default und Help kommen direkt aus der Funktionssignatur und dem Docstring. Kein extra Parser-Code.

    typer.Argument für Positionsargumente (müssen in fester Reihenfolge übergeben werden), typer.Option für Flags (--name wert). Kurzformen wie -f lassen sich mitgeben:

    def kopieren(
    quelle: str = typer.Argument(..., help="Quelldatei"),
    ziel: str = typer.Argument(..., help="Zieldatei"),
    force: bool = typer.Option(False, "--force", "-f"),
):
    ...
# Aufruf: python tool.py a.txt b.txt -f

    Pflichtfeld: Default auf ... setzen (statt eines Wertes).

    def run(rate: float = typer.Option(...)):  # rate muss übergeben werden
    ...
# Fehlt rate beim Aufruf, bricht typer mit Fehlermeldung ab.

    Optionale Werte und Listen über Type Hints:

    def run(
    name: Optional[str] = None,           # darf weggelassen werden
    tags: list[str] = typer.Option([]),   # --tags a --tags b
):
    ...

    Auswahl über Enum – typer macht daraus automatisch --modus dev|prod|test:

    class Modus(str, Enum):
    dev = "dev"
    prod = "prod"

def run(modus: Modus = Modus.dev): ...

    Pfade werden automatisch geprüft (existiert die Datei? lesbar? Datei oder Ordner?):

    def analyze(
    datei: Path = typer.Argument(..., exists=True, readable=True,
                                 file_okay=True, dir_okay=False),
):
    ...
# Wird nichts gefunden, bricht typer ab — kein eigener Check nötig.

    Eigene Validierung über Callback (z. B. nur positive Zahlen):

    def positive(value: int) -> int:
    if value <= 0:
        raise typer.BadParameter(f"{value} muss positiv sein")
    return value

def run(threads: int = typer.Option(4, callback=positive)):
    ...

    Eingabe abfragen, auch mit Bestätigung (z. B. Passwort doppelt eingeben):

    def login(
    name: str = typer.Option(..., prompt=True),
    passwort: str = typer.Option(..., prompt=True, hide_input=True,
                                 confirmation_prompt=True),
    api_key: str = typer.Option(..., envvar="MY_API_KEY"),
):
    ...

    Subcommands: mehrere @app.command() an derselben App. Verschachtelt mit add_typer:

    app = typer.Typer()
users_app = typer.Typer()
app.add_typer(users_app, name="users")

@users_app.command("add")
def users_add(name: str): ...
# Aufruf: python tool.py users add alice

    Shell-Completion für bash, zsh, fish, PowerShell:

    $ python tool.py --install-completion

    Help-Texte und Fehler werden mit rich formatiert (Farben, Tabellen) – ohne extra Aufwand.

    Befehl gezielt mit Exit-Code beenden (z. B. bei Fehler):

    if not datei.exists():
    typer.echo("Datei fehlt", err=True)
    raise typer.Exit(code=1)

    Vorteile - Type Hints liefern Typ, Default und Validierung in einem - IDE-Autocomplete und mypy funktionieren nativ - Erbt click-Features (Prompts, Farben, Subcommands, Tests) - Shell-Completion ohne Aufwand

    Nachteile - Bei komplexen Fällen muss man trotzdem click kennen (da es darauf aufbaut) - Zwei API-Stile mischen sich (Funktionssignatur + typer.Option(...))

    rich (Ergänzung)¶

    Kein CLI-Parser, sondern Bibliothek für schöne Terminal-Ausgaben (Farben, Tabellen, Progressbars). Bei typer schon mit dabei.

    Installation: pip install rich · https://github.com/textualize/rich

    from rich import print
from rich.progress import track

print("[bold red]Fehler![/bold red] Datei [green]nicht gefunden[/green]")

for _ in track(range(100), description="Verarbeite..."):
    ...

    Weitere nützliche Features:

    Tabellen (z. B. Sync-Status pro Datei):

    from rich.table import Table
t = Table(title="Status")
t.add_column("Datei"); t.add_column("Zustand")
t.add_row("notes.txt", "synced")
print(t)

    Ordner-/Dateibaum:

    from rich.tree import Tree
tree = Tree("Dokumente")
tree.add("notes.txt")
tree.add("bilder/").add("foto.jpg")
print(tree)

    Syntax-Highlighting (z. B. YAML-Konfig anzeigen):

    from rich.syntax import Syntax
print(Syntax(open("config.yaml").read(), "yaml"))

    Logs mit Zeitstempel:

    from rich.console import Console
Console().log("Sync gestartet")
# [12:00:01] Sync gestartet  (mit Datei + Zeile)

    Lesbarere Fehler (Quellcode mit Highlight statt rohem Traceback):

    from rich.traceback import install
install()  # einmal beim Start aufrufen
